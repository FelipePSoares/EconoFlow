# econoflow-mobile

React Native 0.85 + Expo SDK 56 app. Read the exact versioned docs at https://docs.expo.dev/versions/v56.0.0/ before writing any code.

## Build (release APK)

```powershell
cd econoflow-mobile

# 1. Regenerate native Android project (do this when deps change or android/ is stale)
$env:EXPO_PUBLIC_API_URL = "https://econoflow.pt"
npx expo prebuild --platform android --no-install --clean

# 2. Generate a debug signing keystore (one-time)
keytool -genkey -v `
  -keystore "$env:USERPROFILE\.android\release.keystore" `
  -storepass android -alias econoflow -keypass android `
  -keyalg RSA -keysize 2048 -validity 10000 `
  -dname "CN=EconoFlow,O=EconoFlow,C=PT"

# 3. Build release APK
$env:JAVA_HOME = "C:\tools\jdk-17.0.13+11"
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
$env:EXPO_PUBLIC_API_URL = "https://econoflow.pt"
$keystore = "$env:USERPROFILE\.android\release.keystore"

cd android
.\gradlew assembleRelease `
  "--project-prop=android.injected.signing.store.file=$keystore" `
  "--project-prop=android.injected.signing.store.password=android" `
  "--project-prop=android.injected.signing.key.alias=econoflow" `
  "--project-prop=android.injected.signing.key.password=android" `
  --no-daemon

# APK will be at: android/app/build/outputs/apk/release/app-release.apk
```

## Dev on emulator with local API (localhost:7003)

To run the app on the emulator pointing to the local backend, three things must be configured:

### 1. Backend must listen on all interfaces (0.0.0.0)

The Android emulator reaches the host at `10.0.2.2` (NOT localhost). The backend's Kestrel config in `appsettings.Development.json` needs an explicit endpoint binding to `0.0.0.0`:

```json
"Kestrel": {
  "Endpoints": {
    "Https": {
      "Url": "https://0.0.0.0:7003"
    }
  },
  "Certificates": {
    "Default": {
      "Path": "C:\\Users\\felip\\.econoflow\\devcert-emulator.pfx",
      "Password": "econoflow"
    }
  }
}
```

**IMPORTANT**: The cert must NOT have `basicConstraints: CA:TRUE` — Android rejects CA certificates used as server certs. Use `devcert-emulator.pfx` (a self-signed end-entity cert generated via PowerShell's `CertificateRequest.CreateSelfSigned`) which has the required SANs (`10.0.2.2`, `127.0.0.1`, `localhost`) and `serverAuth` EKU, but no CA constraint.

The original `devcert.pfx` (generated by `@expo/devcert`) has `CA:TRUE` — this works fine for browsers that don't enforce the constraint, but Android's TLS stack does enforce it.

### 2. Emulator must trust the dev cert

Both the old CA cert (`c406ed6d.0`) and the new self-signed cert (`f191a36c.0`) must be injected into the emulator's system trust store:

```powershell
adb root
adb push $env:USERPROFILE\.econoflow\c406ed6d.0 /data/local/tmp/c406ed6d.0
adb push $env:USERPROFILE\.econoflow\devcert-emulator.pem /data/local/tmp/f191a36c.0
# Quick injection (survives until next reboot):
adb shell "
  mkdir -p /data/local/tmp/ca-overlay
  cp /apex/com.android.conscrypt/cacerts/* /data/local/tmp/ca-overlay/
  cp /data/local/tmp/c406ed6d.0 /data/local/tmp/ca-overlay/
  cp /data/local/tmp/f191a36c.0 /data/local/tmp/ca-overlay/
  mount --bind /data/local/tmp/ca-overlay /apex/com.android.conscrypt/cacerts
  mount --bind /data/local/tmp/ca-overlay /system/etc/security/cacerts 2>/dev/null
"
```

The cert's Android subject hash (`f191a36c`) is the MD5 of the DER-encoded canonical Subject, first 4 bytes reversed (little-endian uint32). This was computed manually with PowerShell.

### 3. Network security config

Android apps must declare a `network_security_config.xml` to allow both cleartext (Metro dev server on port 8081) and HTTPS to `10.0.2.2`:

```xml
<!-- android/app/src/main/res/xml/network_security_config.xml -->
<network-security-config>
  <domain-config cleartextTrafficPermitted="true">
    <domain includeSubdomains="false">10.0.2.2</domain>
    <trust-anchors>
      <certificates src="system" />
      <certificates src="user" />
    </trust-anchors>
  </domain-config>
</network-security-config>
```

Reference it in `AndroidManifest.xml`:
```xml
<application android:networkSecurityConfig="@xml/network_security_config" ...>
```

Note: `npx expo prebuild --clean` wipes the `android/` directory, so both `network_security_config.xml` and the `AndroidManifest.xml` change must be re-applied after prebuild.

### 4. Set API URL in .env.local

```env
EXPO_PUBLIC_API_URL=https://10.0.2.2:7003
```

This is picked up by Metro at bundle time (debug builds) and embedded by Gradle at build time (release builds). The fallback in `src/api/client.ts` is `https://localhost:7003` — fine for web but wrong for emulator.

### 5. Build only for emulator ABI (x86_64)

Set `reactNativeArchitectures=x86_64` in `android/gradle.properties` to avoid building for 4 architectures (build goes from ~15min to ~2min). Also add `"abiFilters": ["x86_64"]` to `app.json` under `expo.android` so future prebuilds pick it up.

### 6. AVOID: Restarting ADB while app install is in progress

If `adb install` is interrupted (timeout, kill-server), the package becomes **frozen** (`SecurityException: Package is currently frozen`). The only fix is to cold-boot the emulator. Always give `adb install` enough time (the APK is ~56MB).

### 7. Cert summary

| File | Type | SAN | CA:TRUE? | Use |
|------|------|-----|----------|-----|
| `~/.econoflow/devcert.pfx` | Self-signed CA | `localhost`, `10.0.2.2`, `127.0.0.1` | Yes | **Do NOT use for Kestrel** (Android rejects) |
| `~/.econoflow/devcert-emulator.pfx` | Self-signed end-entity | `localhost`, `10.0.2.2`, `127.0.0.1` | No | Use in Kestrel for emulator dev |
| `~/.econoflow/c406ed6d.0` | CA public key (PEM) | — | — | Trust anchor in emulator |
| `~/.econoflow/devcert-emulator.pem` | End-entity public key (PEM) | — | — | Trust anchor in emulator |

### Complete dev workflow

```powershell
# Terminal 1: Start backend (listens on all interfaces)
dotnet run --project ./EasyFinance.Server

# Terminal 2: Start emulator
& "$env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe" -avd Pixel_7_API_35 -no-boot-anim

# Terminal 2: Inject certs (after each emulator cold boot)
adb root
adb push $env:USERPROFILE\.econoflow\c406ed6d.0 /data/local/tmp/c406ed6d.0
adb push $env:USERPROFILE\.econoflow\devcert-emulator.pem /data/local/tmp/f191a36c.0
adb shell "
  mkdir -p /data/local/tmp/ca-overlay
  cp /apex/com.android.conscrypt/cacerts/* /data/local/tmp/ca-overlay/
  cp /data/local/tmp/c406ed6d.0 /data/local/tmp/ca-overlay/
  cp /data/local/tmp/f191a36c.0 /data/local/tmp/ca-overlay/
  mount --bind /data/local/tmp/ca-overlay /apex/com.android.conscrypt/cacerts
  mount --bind /data/local/tmp/ca-overlay /system/etc/security/cacerts 2>/dev/null
"

# Terminal 2: Build & run on emulator
cd econoflow-mobile
$env:EXPO_PUBLIC_API_URL = "https://10.0.2.2:7003"
npx expo run:android
```

## Key patterns

- **API client**: `src/api/client.ts` � Axios instance with token refresh interceptor
- **Env var for API URL**: `EXPO_PUBLIC_API_URL` (default: `https://econoflow.pt`, must be set **without** `/api` suffix since all endpoints start with `/api/`)
- **State**: Zustand stores in `src/store/` for auth & project; React Query for server state
- **Navigation**: `@react-navigation/native-stack` + `bottom-tabs` in `src/navigation/`
- **i18n**: `react-i18next` with JSON files in `src/i18n/locales/`
- **Theme hook**: `useAppTheme()` from `src/theme/useAppTheme.ts` � wraps Paper's `useTheme` with custom colour types
- **Error handling**: `ErrorBanner` component exists at `src/components/common/ErrorBanner.tsx`
